# 작업 종료 절차

사용자가 "작업종료하자", "작업 끝", "마무리하자" 등을 말하면 아래 절차를 **자동으로** 수행합니다.

## 1단계: 작업 내용 정리

오늘 대화에서 수행한 모든 작업을 분류합니다:
- **새 기능 추가** → 업그레이드 히스토리에 기록
- **버그 수정** → 버그수정 히스토리에 기록
- **리팩토링/개선** → 업그레이드 히스토리에 기록
- **설정 변경** → 작업일지에 기록

## 2단계: 프로젝트 문서 업데이트

### CHANGELOG.md (프로젝트 루트)
// turbo
```
프로젝트 루트의 CHANGELOG.md에 오늘 날짜로 변경사항 추가
```

형식:
```markdown
## [날짜] - 작업 요약

### ✨ 새 기능
- 기능 설명

### 🐛 버그 수정
- 수정 내용

### 🔧 개선/리팩토링
- 개선 내용
```

### 작업일지 (docs/dev-log/ 또는 대시보드 페이지)
프로젝트에 작업일지 페이지가 있으면 해당 페이지의 데이터에도 추가합니다.

### 업그레이드 히스토리 / 버그수정 히스토리
프로젝트 대시보드에 해당 메뉴가 구현되어 있으면 DB나 데이터 파일에도 기록합니다.

## 3단계: 코드 커밋 & 푸시

1. 변경되지 않은 파일 확인: `git status`
2. 문서 변경사항 포함하여 커밋
```
git add -A
git commit -m "docs: 작업일지 및 변경 히스토리 업데이트 (YYYY-MM-DD)"
```
3. 푸시
```
git push origin main
```

## 4단계: 작업 요약 보고

사용자에게 오늘 작업 요약을 간결하게 보고합니다:
- 총 변경 파일 수
- 주요 변경 사항 (3줄 이내)
- 배포 상태
- 다음 세션에서 할 일 (있다면)

## 참고
- CHANGELOG.md가 없으면 새로 생성합니다
- docs/ 폴더가 없으면 생성합니다
- 빌드 에러가 있으면 문서만 커밋합니다 (코드는 이미 이전에 커밋된 상태)
